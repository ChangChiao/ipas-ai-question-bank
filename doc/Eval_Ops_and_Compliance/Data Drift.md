# Data Drift (數據漂移)

> **📝 概念解釋：**
> 資料漂移 (Data Drift / Covariate Shift) 指的是當模型實際上線運作後，真實世界產生的「輸入資料分佈特徵」與當初用來訓練模型的「歷史資料分佈」產生了顯著的偏離。這是一種模型退化(Model Decay)的現象，當輸入的基礎發生了變化，即使模型演算法沒有改，其預測的準確度也會隨之崩盤與崩潰。
>
> **💡 使用場景：**
> 在電商推薦系統中，可能發生消費者品味突然改變；或是房價預測模型在上線半年後遇到了不可預期的經濟泡沫，導致真實房市數據(輸入)與半年前的歷史訓練數據脫節。這時 MLOps 團隊就需監控 Data Drift 並即時重新訓練微調模型。
>
