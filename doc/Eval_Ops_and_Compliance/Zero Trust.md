# Zero Trust (零信任架構)

> **📝 概念解釋：**
> 零信任架構 (Zero Trust Architecture) 是一種顛覆傳統「內網安全」概念的資安模型。它的核心理念為「永遠不信任，始終要驗證 (Never Trust, Always Verify)」。無論請求來自於企業外部還是已經連入內部網路的終端設備，系統都會假設網路環境已經被入侵，因此針對每一個資料存取行為，都必須進行嚴格的身分認證、並落實「最小權限原則」。
>
> **💡 使用場景：**
> 防範內部資料洩漏，特別是企業建立專屬私有 LLM 知識庫時，必須防止一位只擁有研發部權限的員工，透過 prompt injection 的手段偷問知識庫去存取不屬於他層級的財務機密檔案。這時就必須從基礎網路架構實行身分驗證。
>
> **🔗 關聯關鍵字：**
> `Cybersecurity`, `Authentication`, `Least Privilege`, `IAM`
