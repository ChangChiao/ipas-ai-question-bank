# Model Sizes (7B, 70B)

## 📝 概念解釋 (Concept Explanation)
在討論大型語言模型 (LLM) 時，常常會看到名稱後面跟著「7B」、「70B」甚至「405B」的後綴。這裡的「B」代表的是十億 (Billion)，而前方的數字則是指該模型內部包含的「參數數量 (Parameters Number)」。
參數數量就像是大腦中的神經元突觸連結數量，它是衡量模型複雜度、儲存知識容量以及邏輯推理能力的重要指標。
- **7B 到 8B 級距 (小模型)**：相當於 70 億到 80 億個參數。這類模型雖然體積小，但是在單一特定任務上表現不錯，而且佔用的記憶體 (VRAM) 較少，一般家用高階顯卡或是手機端點都跑得動。
- **70B 以上級距 (大模型)**：相當於 700 億甚至幾千億個參數。這類模型具備極強的世界常識、多語言翻譯能力、複雜邏輯推導以及編寫程式的能力，但缺點是需要動用多張高昂的資料中心等級 GPU (如 A100/H100) 才能運行。

## 💡 使用場景 (Use Cases)
- **邊緣運算與成本敏感型服務**：一家企業想要在客服系統部署自動回覆 AI，為了節省雲端長期的運算開銷和提高回覆速度，選擇了 Llama-3-8B（80億參數），這樣只要一台普通的伺服器就能滿足全天的需求。
- **超級大腦與全能助理**：研究機構需要一個能讀懂數學論文並且幫忙找 BUG 的 AI，他們會毫不猶豫選擇 70B 甚至 400B 規模以上的模型，這時需要龐大的算力資源，但能換來媲美頂尖專家的推理建議。
