# Chunking (文本分塊)

> **📝 概念解釋：**
> 切塊或分塊 (Chunking) 是一種前置的資料處理技術。由於 LLM 與嵌入向量模型 (Embedding Models) 有長度限制（如 512 或 8192 個 Token），我們無法直接將整本上百頁的 PDF 書籍塞給模型運算。Chunking 的目的就是把長篇文章依據固定字數限制（例如每 500 字一塊）、重疊字數（Overlap）、或者是按照文章自然的段落標題或標點符號進行「合理地切割成多個碎片」，確保每一小節保有完整的最小語意。
>
> **💡 使用場景：**
> 建置 RAG（檢索增強生成）系統的第一道重要工序。切塊切得好，後續的向量搜尋(Semantic Search)才能準確找到使用者需要的知識；若切得太小或切斷了因果關係句子，會直接導致 RAG 查詢失敗。
>
> **🔗 關聯關鍵字：**
