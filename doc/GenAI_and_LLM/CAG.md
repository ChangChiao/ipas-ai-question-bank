# CAG (Cache-Augmented Generation)

> **📝 概念解釋：**
> 快取增強生成 (Cache-Augmented Generation, CAG) 是一種新興的 LLM 推理最佳化技術設計。隨著大型語言模型支援的上下文視窗 (Context Window) 變得極大，每次處理長達十萬的提詞會極度吃資源。CAG 取代了傳統 RAG 把大量外部切片資料「塞進問題去問」的時間消耗，它事先對海量擴充文件預先算出並長期保存 KV Cache (Key-Value Cache)。當使用者問問題時，LLM 就不需重新閱讀文件，直接「套用」快取過的記憶狀態來進行推理生成。大語言模型藉此實現超長文本的無延遲回應。
>
> **💡 使用場景：**
> 特別適合在「知識庫內容變動不大，且文件極度冗長」的情境下替換 RAG。例如律師事務所可以事先把某大財團幾百頁的歷年財報全部利用 CAG 讀取成快取，之後所有律師詢問財報相關內容的回應速度將會大幅躍升，且不需要向量資料庫的檢索動作。
>
> **🔗 關聯關鍵字：**
> `KV Cache`, `RAG`, `Context Window`, `LLM Inference`
